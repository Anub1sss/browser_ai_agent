## Task Hunter

CLI-инструмент для запуска веб-задач с профилями, очередью и автоматизацией браузера.

### Возможности
- запуск одиночных задач и пакетных очередей
- приоритеты, ретраи, пауза/продолжение очереди
- профили запуска (провайдер, модель, задержки, прокси)
- резервная модель на случай ошибок
- под‑агент планирования перед выполнением
- постоянные сессии для сохранения логинов и куки

### Быстрый старт
```bash
cd /Users/macbook/Downloads/browser-ai
source "$HOME/.local/bin/env"
source .venv/bin/activate
python3 task_hunter.py
```

### Настройка
1) Скопируйте `.env.example` в `.env` и заполните ключи провайдеров.
2) Запустите `python3 task_hunter.py` и создайте профиль в меню.

### Переменные окружения
- `OPENAI_API_KEY` — ключ OpenAI (если используете OpenAI).
- `ZAI_API_KEY` — ключ GLM (если используете GLM).

### Меню
- `Run single task` — выполнить одну задачу (можно изменить параметры запуска).
- `Batch queue` — очередь задач с приоритетами, ретраями и паузой.
- `Manage profiles` — управление профилями запуска.

### Параметры запуска (в профиле и при запуске)
- `Provider` — провайдер LLM (custom/openai/glm).
- `Model` — модель провайдера.
- `Persistent session` — сохранять профиль браузера между запусками.
- `Fallback` — запасной LLM при ошибках основного.
- `Sub-agent` — сначала строит план, затем выполняет.
- `Wait between actions` — задержка между действиями.
- `Start delay` — задержка перед началом действий.
- `Proxy` — прокси настройки (server/bypass/username/password).

### Примеры задач
- «Найди топ‑3 популярных блюда в Москве и сохрани их в заметку».
- «Сравни цены на ноутбуки в трех магазинах».

### Где лежат данные
Профили и очередь сохраняются в `~/.task_hunter/`:
- `profiles.json`
- `queue.json`
